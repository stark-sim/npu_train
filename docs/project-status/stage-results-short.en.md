# Interim Project Results Summary (Short Version)

The key achievement of this project is not that it defines an entirely new parallel training framework, but that it successfully enables a practical large-model training path under the constraints of `Ascend 910A + lower CANN + PyTorch/torch_npu`. The work completed so far includes a tensor-parallel training foundation for Ascend NPUs, dedicated adaptation for DeepSeek-V2-Lite and MoE architectures, engineering workarounds for NPU-incompatible operators, and an end-to-end workflow for real-data training, validation, and checkpoint resume.

At the same time, it should be stated clearly that Huawei’s official MindSpore / MindSpore Transformers stack has already covered many general-purpose capabilities, including TP, PP, MoE training, DeepSeek mainline model support, weight conversion, dataset handling, and memory optimization. From the perspective of general capability coverage, part of our work is no longer addressing an official blank area.

However, that does not make the current work meaningless. The reason is that the official newer framework primarily targets newer platforms and higher software versions, while our actual environment remains `910A + lower CANN + PyTorch/torch_npu`. Under this constraint, the fact that the official ecosystem “has the capability” does not mean it can directly replace what currently works in our environment. The real value of this project lies in practical adaptation for a legacy platform: making DeepSeek-V2-Lite, MoE, HCCL communication compatibility, NPU operator workarounds, and real-data training actually run and remain stable.

Therefore, the most accurate positioning of the current results is: **the project has enabled the critical training path for large models under legacy-platform constraints and produced a runnable, reproducible, and extensible engineering solution; its main value lies in platform adaptation and practical delivery rather than in general framework capability itself.**
